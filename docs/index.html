podcast-members-feed

Self-hosted Bitcoin-native podcast membership — BTCPay Server subscriptions, private RSS feeds, and a token service that bridges the two.

Deployment Management CLI Monitoring Technical Reference GitHub

How it works

When a listener subscribes through BTCPay Server, the token service receives a signed webhook, issues a unique private feed URL, and delivers it via email or Nostr DM. The subscriber adds that URL to any podcast app and gets access to member-only episodes. When the subscription expires or is cancelled, the feed URL stops working automatically. 

BTCPay Server
    │ HMAC-signed webhook
    ▼
podcast-token-service
    │                  │               │
    ▼                  ▼               ▼
 SQLite DB      PodServer feed    Nostr relays / SMTP
(tokens.db)    (upstream proxy)   (notifications)

Deployment

Path A — Umbrel + VPS

BTCPay, LND, and Alby Hub run on Umbrel. The token service runs as a Podman container on a small VPS connected via Tailscale. 

git clone https://github.com/genebean/podcast-members-feed.git
cd podcast-members-feed/podman
cp .env.example .env
# Edit .env with your values
podman pull ghcr.io/genebean/podcast-members-feed:latest
podman compose up -d

Path B — NixOS (fully declarative)

Add as a flake input:

inputs = {
  podcast-members-feed.url = "github:genebean/podcast-members-feed";
  nixpkgs.follows          = "podcast-members-feed/nixpkgs";
};

Import the module and enable the service:

imports = [ podcast-members-feed.nixosModules.podcast-token-service ];

services.podcastTokenService = {
  enable          = true;
  package         = pkgs.podcast-token-service;
  environmentFile = config.sops.secrets."podcast-token-service-env".path;
};

See nixos-configurations/example-host/configuration.nix for a complete example including nix-bitcoin, Alby Hub, nginx, TLS, Tailscale, and sops-nix.

Environment variables

VariableRequiredDescription
BTCPAY_WEBHOOK_SECRET✓HMAC-SHA256 shared secret from BTCPay webhook settings
PODSERVER_FEED_URL✓Internal URL of the PodServer members RSS feed
FEED_BASE_URL✓Public base URL for subscriber feed URLs and /recover
ADMIN_TOKEN✓Bearer token for /metrics and /admin/ endpoints
NOSTR_PRIVATE_KEY✓Service Nostr keypair: nsec bech32 or 32-byte hex
DATABASE_PATHPath to SQLite database (default: tokens.db)
EXPIRED_AUDIO_URLURL of pre-recorded expiry audio clip (optional)
SMTP_HOSTSMTP server (omit to disable email)
SMTP_PORTSMTP port (default: 587)
SMTP_USERSMTP username
SMTP_PASSWORDSMTP password
SMTP_FROMSender address for outgoing email

Management CLI

The podcast-members-manage command reads the SQLite database directly and is installed alongside the service binary.

CommandDescription
statsPrint counts: total subscribers, active tokens, never-accessed, expired-unnotified
subscribersList subscribers. Flags: --active, --never-accessed, --expiring-days N
feed-urlPrint feed URL for a subscriber. Lookup by --email or --npub
revoke <id>Revoke all tokens for a BTCPay subscriber ID (prompts unless --yes)
cleanupPOST to /admin/cleanup on the running service (requires ADMIN_TOKEN)
test-webhookEnd-to-end service validation: health, metrics, webhook, feed proxy, expiry flow

Example usage — NixOS

podcast-members-manage stats
podcast-members-manage subscribers --active
podcast-members-manage subscribers --expiring-days 7
podcast-members-manage feed-url --email subscriber@example.com
podcast-members-manage revoke <btcpay-subscriber-id>

Example usage — Podman (Path A)

podman exec podcast-token-service \
  podcast-members-manage \
  --db /var/lib/podcast-token-service/tokens.db \
  stats

Monitoring

Prometheus metrics are available at /metrics with bearer token auth. AlertManager rules are in alerts/podcast-members-feed.rules.yml. 

# prometheus.yml scrape config
- job_name: podcast_token_service
  bearer_token: YOUR_ADMIN_TOKEN
  static_configs:
    - targets: ["members.yourpodcast.com"]
  scheme: https

Building the container locally

nix build .#dockerImage
podman load < result
podman run --rm -p 8765:8765 \
  --env-file podman/.env \
  localhost/podcast-token-service:latest
